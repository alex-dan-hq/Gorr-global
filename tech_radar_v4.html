Licensing Matters
Apache 2.0 and MIT = safe. Everything else has gotchas.
Safe: Apache 2.0 (Qwen, Mistral) — full freedom + patent grant + EU AI Act exemptions. MIT (DeepSeek, Phi) — maximally permissive, no patent grant. Risk: Llama (EU ban, MAU cap). Decision Framework
<1B tokens/month: Any API cheaper than self-hosting. 1–10B: Hosted OS APIs optimal (Together, Groq). >10B with MLOps: Self-hosting (vLLM/SGLang) may win. Air-gapped: Self-hosted OS only. Consensus: start closed → validate → migrate high-volume to OS. Build abstraction layers from day one. Self-hosting breakeven: 50K+ daily requests with predictable load. Startup first-year self-hosting: $350K–$500K.
Cost Optimisation Stack
Hardware era shift: B200/GB200 in volume production (Feb 5, TSMC Arizona). 25× energy efficiency per token. FP4 precision: 15–30× throughput. GPU cloud pricing: H100: $2.72–3.50/hr (down 64–75%); H200: $2.50–4.31/hr; B200: $5.58/hr (limited). Quantization: INT4/FP4 = 60–70% cost reduction. MoE: Llama 4 Scout 17B/109B = frontier at fraction of compute. Self-hosting breakeven: ~200M+ tokens/day (8B) or ~70M+ (70B).

Voice Stack Insight
STT is commodity for English. Multilingual STT is functional. Real-time conversation is GA but fragile.
Open source (NVIDIA Canary, CosyVoice2) rivals commercial for accuracy. Key gap: accented English + noisy environments (retail floors, warehouses) — no benchmark data, needs real-world testing. For retail workforce: voice-first interface is technically viable today for English, near-viable for Polish/Romanian/Hindi with ~14.5% WER streaming. Real-time voice conversation (bidirectional) still has session quality issues and latency — better for short interactions than long coaching sessions.

2026 Consensus
RAG for volatile facts and large knowledge bases. Long context for holistic analysis under ~200K tokens (now cheaper with flat pricing). Fine-tuning for stable behaviours. 40–60% of RAG implementations still fail to reach production due to retrieval quality and governance. The architecture choice is no longer "RAG vs long context" — it's "which combination for which data type."
Emerging
Multimodal embeddings change retrieval fundamentally
Gemini Embedding 2: one model, one vector space for all modalities. Previously: separate embedding models, separate indices, complex fusion. This enables "search everything" — text, photos, audio, video, PDFs in one query. Brand new, no production track record. But architecturally significant: if it works, it simplifies the entire retrieval stack.
